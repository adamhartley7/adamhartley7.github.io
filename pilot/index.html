Forecast first. Record the real result after.

Four coded participant slots complete six usable tasks each. Each person uses at least three task classes. Slots are pseudonymous, not anonymous, when the coordinator can link a slot to a person. This is an instrumentation check, not evidence that TOP is accurate.

This page never asks for the task's title, description, prompt, reply, or files. No task text, names, prompts, replies, file contents, paths, credentials, account identifiers, or notes. The task-text-free file still retains the coded slot, export date, task class, forecast version, P10, P50, P90, actual, attempt state and sequence, invalidation reason, and completeness counts. The pilot makes no runtime request that transmits participant data.

How to read this

  • The P10-to-P90 interval is an experimental research output. This public page does not publish a performance rate.
  • The underlying research export preserves the paired records for analysis outside this public interface.
  • The USD values are API-rate equivalents, not subscription bills.
  • Multiplicative error is 1.0 at a perfect P50 forecast. Lower is better.
  • Mean log bias is positive when P50 tends to be too high, and negative when it tends to be too low.
  • The log-space interval score rewards narrow intervals but penalizes misses. Lower is better.
  • Missing actuals and every excluded forecast remain visible in the data-completeness counts.
  • Forecast-version splits are descriptive. This tool does not randomize or balance assignments and cannot establish causal differences between versions.
